# MessageQueue Demo (自定义协议消息队列)

本项目是一个轻量级的消息队列（Message Queue）中间件，模拟了经典消息队列（如 RabbitMQ）的核心架构模式。项目完全采用 Java 编写，基于自定义的 TCP 通信协议进行数据交互。

## 项目核心功能与实现原理

本项目分为两大核心模块：**BrokerServer（服务端）** 和 **MQClient（客户端）**。

### 1. 服务端核心设计 (BrokerServer)
服务端主要负责消息的接收、路由、持久化以及与客户端的网络通信逻辑。

*   **四大核心实体**：
    *   **Exchange (交换机)**：负责接收生产者发送的消息，并根据路由规则将消息投递到绑定的队列中。支持 Direct（直接）、Fanout（扇出）、Topic（主题）等核心路由策略。
    *   **Queue (队列)**：用于暂存待消费的消息。
    *   **Binding (绑定关系)**：维护 Exchange 与 Queue 之间的映射关联，以及路由比对所需要的 RoutingKey。
    *   **Message (消息体)**：流转在系统中的核心数据载体，包含路由属性数据与支持传输任何格式的二进制业务载荷。
*   **混合数据持久化层 (DataCenter)**：
    *   **元数据存储 (DataBaseManager)**：采用 SQLite 轻量本地数据库，借助 MyBatis 框架实现交换机、队列、绑定等相对静态元数据的持久化存储管理。
    *   **消息文件存储 (MessageFileManager)**：将消息的二进制数据以顺序追加的形式高速写入本地磁盘文件，同时记录消息在文件中的区间偏移量，极大幅度优化 IO 性能。
    *   **内存数据中心 (MemoryDataCenter)**：采用 ConcurrentHashMap 在服务端内存中全量缓存实时数据结构，在高并发场景下提供微秒级的寻址与转发能力。
*   **网络通信与多路复用**：
    *   基于原生 ServerSocket (TCP) 实现服务器底层的长连接监听。
    *   通过解析客户端统一规范的数据包进行业务分发。使用 VirtualHost 的概念实现资源的逻辑隔离。

### 2. 客户端核心设计 (MQClient)
客户端为上层应用屏蔽了底层的通信细节，提供了语义一致且易用的编程接口。

*   **通信核心组件**：
    *   **ConnectionFactory (连接工厂)**：用于加载服务端配置并获取网络连接。
    *   **Connection (连接实体)**：持有与 BrokerServer 间的单根物理 TCP 长连接，减少网络握手开销。
    *   **Channel (信道)**：在单一物理 Connection 上复用的多条逻辑通道。所有的队列声明、消息发送操作均在 Channel 层面按序列化指令完成，大幅度减少资源的消耗。
*   **消费者模型 (Consumer)**：
    *   提供异步监听的 Push 模型。当 BrokerServer 将匹配的消息推送到客户端时，内置专门的线程池负责反序列化消息体系，触发消费者的回调业务逻辑代码并自动或手动完成消息接收校验（ACK）。

## 项目源码结构说明

项目在代码层面根据不同职责进行了清晰的层级划分：

*   **`mqserver.core`**：存放上述四大基本实体（交换机、队列、消息、绑定）的数据模型与管理器类。
*   **`mqserver.datacenter`**：核心的数据管理层实现地，整合 SQLite 本地库接口、文件高速读写操作和内存并发表控制。
*   **`mqserver.mapper`**：封装向 SQLite 提交的建表与增删改查 SQL 声明（XML 映射）。
*   **`mqclient.client`**：网络客户端包装器，管理物理连接(Connection)与多路复用连接(Channel)的存活状态。
*   **`mqclient.consumer`**：客户端中接收到指令后的消费者环境配置支持和具体分发执行逻辑。
*   **`common`**：系统级别公共组件。包含枚举等常量表、自定义全局异常类，以及在网络传输中用于二进制转化、JSON序列化的核心工具集。
*   **`demo`**：对外提供的基础示例模块，可以直接执行里面的代码体验生产者发送数据与消费者实时拉取数据的实际效果。
